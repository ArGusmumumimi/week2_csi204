# Analysis & Design - Hotel Booking System

## 1. ผู้เกี่ยวข้องกับระบบ (Actors)

1. **Customer** - ลูกค้าที่จองห้องพักผ่านออนไลน์
2. **Receptionist** - พนักงานต้อนรับ
3. **Manager** - หัวหน้าฝ่าย
4. **System** - ระบบอัตโนมัติ (ส่งอีเมล/แจ้งเตือน)

## 2. Use Case ตาม Actor

### 2.1 Customer
- ค้นหาห้องพัก (ตามวันที่ / ประเภทห้อง)
- ดูรายละเอียดห้องพักและราคา
- จองห้องพัก (ระบุวันที่, จำนวนผู้เข้าพัก, แพ็กเกจเสริม)
- ชำระเงิน (บัตรเครดิต/เดบิต, โอนเงิน, พร้อมเพย์)
- ดูประวัติการจอง
- ยกเลิกการจอง

### 2.2 Receptionist
- บันทึกเช็คอิน / เช็คเอาท์
- ตรวจสอบสถานะห้องพัก
- ตรวจสอบข้อมูลการจองล่วงหน้า

### 2.3 Manager
- ดูข้อมูลการจองทั้งหมด/รายบุคคล
- ตรวจสอบข้อมูลการเข้าพัก
- เพิ่ม/แก้ไข/ลบข้อมูลห้องพัก
- ตรวจสอบกรณียกเลิกเกินกำหนดเวลา

### 2.4 System (Automation)
- ส่งอีเมลยืนยันการจอง
- บันทึกและแจ้งเตือนการยกเลิก
- แจ้งเตือน Manager เมื่อยกเลิกเกินกำหนด

## 3. Functional Requirements

| รหัส | ความต้องการ | Actor |
|---|---|---|
| FR-01 | ระบบค้นหาห้องว่างตามวันที่/ประเภท | Customer |
| FR-02 | ระบบจองห้องพักพร้อมเลือกแพ็กเกจเสริม | Customer |
| FR-03 | ระบบชำระเงินหลายช่องทาง | Customer |
| FR-04 | ระบบส่งอีเมลยืนยันอัตโนมัติ | System |
| FR-05 | ระบบยกเลิกการจองตามเงื่อนไข | Customer |
| FR-06 | ระบบบันทึกเช็คอิน/เช็คเอาท์ | Receptionist |
| FR-07 | ระบบจัดการข้อมูลห้องพัก (CRUD) | Manager |
| FR-08 | ระบบแจ้งเตือน Manager กรณียกเลิกเกินกำหนด | System |

## 4. เงื่อนไขทางธุรกิจ (Business Rules)
- ห้องพักปกติรองรับ 2 คน หากเกินต้องมีค่าบริการเพิ่มเติม
- การยกเลิกมีค่าธรรมเนียมตามระยะเวลา (Deadline Policy)
- พนักงานต้อนรับให้ข้อมูลได้ แต่จองแทนลูกค้าไม่ได้
